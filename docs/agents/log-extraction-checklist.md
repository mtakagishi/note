# ログ記事化チェックリスト

AI ログから記事を作る前に、最低限の品質をそろえるためのチェックリスト。

## 使い方

1. `export_sessions.py` で対象ログを取得する。
2. 本チェックリストで抽出内容を確認する。
3. 不足があれば、記事化前に追加質問する。

## 必須チェック

- [ ] 何をしたかったか（目的）が 1 文で言える
- [ ] なぜ調べたか（背景・動機）が明記されている
- [ ] 違和感や困りごとの起点がある
- [ ] 試したことが時系列で整理されている
- [ ] つまずいた点と原因仮説が書ける
- [ ] わかったことと未解決点が分離されている
- [ ] 次に試す行動が 1 つ以上ある

## 事実性チェック

- [ ] 断定文にログ上の根拠がある
- [ ] 推測は推測と明示している
- [ ] 外部情報を使った箇所は出典を確認済み
- [ ] 秘密情報や個人情報が含まれていない

## 読者価値チェック

- [ ] 同じ悩みの読者が検索しそうな語がある
- [ ] 再現可能なコマンドや手順がある
- [ ] 途中経過なら途中経過と明言している
- [ ] 学びが「次にどう使えるか」まで書かれている

## 自己検証プロンプト雛形

```text
以下は AI セッションログから抽出した記事素材です。
次を 1 レスポンスで実行してください。

1) 事実・推測・未確認に3分類する
2) 論理の飛躍がある箇所を5件まで指摘する
3) 読者価値が弱い箇所を3件指摘する
4) 指摘を反映した改稿版アウトラインを提示する

評価観点:
- 事実性
- 納得感
- 検索性
- 再現性
```

## インタビューが必要なサイン

- ログだけでは主題が1つに定まらない
- AIの結論と本人の納得感がズレている
- 公開可否が判断しづらい固有名詞がある
- 読者向けの検索語が不足している
